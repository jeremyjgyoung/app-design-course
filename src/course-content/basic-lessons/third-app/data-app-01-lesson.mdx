## Activity I: Use a Variable

Use the *tomatoEmoji* variable in your code.

To do this add a text component before the the end of the closing View tag. 
Inside the component include the tomatoEmoji within curly braces.

How do you think the app will change?

```jsx title=App.js
// Before
return (
  <View style={styles.container}>
    <TouchableOpacity onPress={updateEmoji} style={styles.button}>
      <Text style={styles.buttonText}>Press This</Text>
    </TouchableOpacity>
    <Text style={styles.emoji}>{emojiData}</Text>
  </View>
);

// After
return (
  <View style={styles.container}>
    <TouchableOpacity onPress={updateEmoji} style={styles.button}>
      <Text style={styles.buttonText}>Press This</Text>
    </TouchableOpacity>
    <Text style={styles.emoji}>{emojiData}</Text>
    <Text>{tomatoEmoji}</Text>
  </View>
);
```

The curly braces {} around tomatoEmoji tell the computer this is JavaScript code.

To extend your learning you can use the tomatoEmoji variable before and after the button and style the emoji.

## Activity II: Change the Initial State

Change the initial state from an empty string to the bread and strawberry emojis.
What do you expect to happen?

Restart the app after making theses changes.

```jsx title=App.js
// Before
let [emojiData, setEmojiData] = useState("");

// After
let [emojiData, setEmojiData] = useState("🍞🥭");
```

The parameter within useState is the initial state.

## Activity III: Change the State After Being Pressed

Change the parameter in the setEmojiData() function from the emoji 🍕 to the word "Pizza".

```jsx title=App.js
// Before
let updateEmoji = () => {
  setEmojiData("🍕");
};

// After
let updateEmoji = () => {
  setEmojiData("Pizza");
};
```

## Activity IV: Rename the Data

Rename emojiData and setEmojiData to foodEmoji and setFoodEmoji.
Make sure to change those variables anywhere they appear.

```jsx title=App.js
// Before
let [emojiData, setEmojiData] = useState("");

// After
let [foodEmoji, setFoodEmoji] = useState("");
```
The names emojiData and setEmojiData are two are variable names for the name of the state and a way to change the state.

Feel free to explore the code here to figure out how useState works.
