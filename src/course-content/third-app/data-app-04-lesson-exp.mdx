# Code Tracing

When trying to understand  new code, looking at blocks of code in order of their use is useful. This technique is called code tracing.

Let's trace the code from Activity Set 3.4. As a reminder here is all of the code from 3.4.

```jsx
import React, { useState } from 'react';
import { TextInput, Text, View, Button, StyleSheet } from 'react-native';

let App = () => {
  let [toDo, setToDo] = useState('');
  let [toDoList, setToDoList] = useState([]);

  let updateCurrToDo = (input) => {
    setToDo(input);
  };

  let updateToDoList = () => {
    setToDoList(toDoList => [...toDoList, toDo]);
  };

  return (
    <View style={styles.container}>
      <TextInput
        style={styles.input}
        onChangeText={updateCurrToDo}
        value={toDo}
      />
      <Button title="Add To Do" onPress={updateToDoList} />
      <View>
        {toDoList.map((toDoItem) => 
          <View style={styles.toDoView}>
            <Text style={styles.toDoText}>{toDoItem}</Text>
          </View>
        )}
      </View>
    </View>
  );
};

let styles = StyleSheet.create({
  container: {
    flex: 1,
    justifyContent: 'flex-start',
    backgroundColor: '#ecf0f1',
    padding: 20,
  },
  input: {
    fontSize: 20,
    height: 40,
    borderColor: 'black',
    borderWidth: 3,
    marginBottom: 10,
    padding: 10,
    backgroundColor: '#ffffff',
  },
  toDoView: {
    marginTop: 20,
    backgroundColor: '#9999bb',
    padding: 10,
  },
  toDoText: {
    fontSize: 20,
  },
});

export default App;
```

The first few lines are *import* statements that bring in useful libraries and the last line with the *export* statement allows the code from this file to be used elsewhere. Finally everything within the styles object relates to the style of the app.

Let's remove the *import* statements, *export* statement, and the styles to look more closely at the code relevant to the functionality of this app.

```jsx

```

Here the *TextInput* component creates a box to input text. Much of the other code here is used to output what you typed in that text input box and hold that data in a variable.

When you type, the *onChangeText* prop runs the *updateCurrToDo* function to update the *toDo* value. Finally, that *toDo* is shown through the *value* prop in the *TextInput* component. Take a moment to see how the code above produces a text input shows what you type and holds the value you have typed in the toDo variable.

Now let's see how when you press Add To Do, that to do is added to your to do list.

```jsx

```

Once you press the Button with the title Add To Do, the *updateToDoList* is run. Then the value typed in the input text box is added to the *toDoList*. The *toDoList* is displayed in below the Add To Do button - before you click on it for the first time the array is empty.
